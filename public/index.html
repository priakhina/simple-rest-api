REST API Documentation

Welcome to the Simple REST API!

This API allows you to manage users, income, and expenses using standard HTTP methods.

All data is stored in a Firebase Realtime Database.

General Endpoint

üì•GET /

Retrieves an overview of the REST API and available endpoints (returns this documentation in HTML format)

Returns a 200 OK status code when successful

Users Endpoints

üì•GET /users

Retrieves a list of all users

Returns a 200 OK status code when successful

Example Response:


            [
                {
                    "address": {
                        "city": "Toronto",
                        "country": "Canada",
                        "street": "123 Main St",
                        "suite": "Apt 4",
                        "zipCode": "M5V 2T6"
                    },
                    "email": "john@example.com",
                    "id": "-OWajfMNuJBa3BGD1FSV",
                    "name": "John Doe",
                    "username": "johndoe"
                }
            ]
üì§POST /users

Creates a new user

All fields shown in the example request below are required

Returns a 201 Created status code when successful

Example Request Body:


            {
                "name": "John Doe",
                "username": "johndoe",
                "email": "john@example.com",
                "address": {
                    "street": "123 Main St",
                    "suite": "Apt 4",
                    "city": "Toronto",
                    "country": "Canada",
                    "zipCode": "M5V 2T6"
                }
            }

Example Response:


            {
                "message": "New user has been created",
                "user": {
                    "id": "-OWajfMNuJBa3BGD1FSV",
                    "name": "John Doe",
                    "username": "johndoe",
                    "email": "john@example.com",
                    "address": {
                        "street": "123 Main St",
                        "suite": "Apt 4",
                        "city": "Toronto",
                        "country": "Canada",
                        "zipCode": "M5V 2T6"
                    }
                }
            }

‚ö†Ô∏è Error Examples

400 Bad Request

Validation error (missing required fields or incorrect data type)

Example response:

When a POST or PUT request is sent and the request body...

misses one required field: 


            {
                "error": "\"address\" is required"
            }

misses multiple required fields: 


            {
            "error": [
                "\"username\" is required",
                "\"email\" is required"
            ]
        }
404 Not Found

Resource doesn't exist

Example Response:

When a GET request is sent and the resource with the provided ID doesn't exist: 


            {
                "error": "User with id -OWajfMNuJBa3BGD1FSV has not been found"
            }

When a GET request is sent to the endpoint that is not recognized/supported by the server (e.g., http://localhost:3001/blahblah): 


            {
                "error": "Unknown endpoint"
            }
409 Conflict

Duplicate resource (email and/or username); applicable for users only

Example Response:

When a POST request is sent to the /users endpoint and a resource with the given username and email already exists: 


            {
                "error": [
                    "A user with username johndoe already exists",
                    "A user with email john@example.com already exists"
                ]
            }
